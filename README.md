# Telegram-бот: новости астрономии, космонавтики, события и фото NASA

Это обновленная версия MVP на Python. Бот умеет:

- присылать новости астрономии и космонавтики из RSS и HTML-лент сайтов;
- сортировать новости по категориям: все, кометы, звездопады/явления, научные открытия, космонавтика;
- подписывать пользователя сразу на несколько категорий;
- показывать события, до которых осталось меньше 7 дней;
- присылать ссылку на трансляцию в момент события, если ссылка добавлена в событие;
- уведомлять о свежих фото NASA APOD и NASA Image of the Day;
- показывать подборку полезных сайтов: карты неба, симуляции, календари, NASA/ESA;
- работать через обычные команды и через кнопки меню.

## 1. Установка

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

В `.env` вставь токен Telegram-бота:

```env
TELEGRAM_BOT_TOKEN=123456:ABC...
NASA_API_KEY=DEMO_KEY
BOT_TIMEZONE=Europe/Moscow
```

Для постоянной работы лучше получить нормальный NASA API key на https://api.nasa.gov/ и заменить `DEMO_KEY`.

## 2. Запуск

```bash
python -m space_astro_bot.main
```

Бот запускается в режиме long polling. Для MVP это проще всего. Для продакшена можно вынести на VPS и завернуть в systemd/Docker.

```text

/news
```

Админ-команды для событий:


Если в `.env` задан `ADMIN_CHAT_IDS`, добавлять и удалять события смогут только эти chat_id. Свой chat_id можно узнать командой:

```text
/myid
```

## 4. Что изменилось в этой доработке

### Кнопки меню

После `/start` появляется обычная клавиатура Telegram:

```text
🚀 Старт / Главное меню
📰 Новости
🌌 События
🖼 Фото NASA
⚙ Настройки
🔗 Полезные сайты
📊 Статус
❓ Помощь
```

Кнопка `🚀 Старт / Главное меню` всегда возвращает пользователя на главный экран. Она нужна, чтобы не искать старую кнопку `Назад` в предыдущих сообщениях.

Также остались inline-кнопки внутри разделов. В настройках и выборе новостей нижняя inline-кнопка `Назад` заменена на `🚀 Старт / Главное меню`.

### Подписка на несколько категорий

Раньше пользователь мог выбрать только одну категорию новостей. Теперь через `/settings` можно выбрать несколько категорий одновременно, например:

```text
Кометы + Звездопады и явления + Космонавтика и миссии
```

Если выбрано `Все новости`, остальные категории не нужны.

### Статус пользователя

Команда `/status` показывает:

- chat_id;
- выбранные категории;
- включены ли автоновости;
- включены ли фото NASA;
- включены ли события;
- часовой пояс;
- количество источников новостей;
- количество пользователей в базе.

### Управление событиями

Добавлены команды:

```text
/listevents
/delevent EVENT_ID
```

`/listevents` показывает ближайшие события и их ID. ID нужен для удаления события.

### Дополнительные источники новостей без изменения кода

В `.env` можно добавить свои RSS-ленты или страницы со списком новостей:

```env
EXTRA_RSS_SOURCES=Название сайта|https://site.com/feed.xml;Другой сайт|https://example.com/news/
```

После изменения `.env` перезапусти бота.

## 5. Как работает сортировка новостей

Файл `space_astro_bot/classifier.py` содержит ключевые слова для категорий:

- `comets` — кометы;
- `meteors` — метеорные потоки, затмения, соединения, сияния и прочие наблюдательные явления;
- `science` — научные открытия, телескопы, галактики, экзопланеты;
- `spaceflight` — ракеты, запуски, миссии, МКС, Artemis, SpaceX.

Это простая, но понятная логика. Позже ее можно заменить на ML/LLM-классификацию или более строгий словарь.

## 6. Источники новостей и фото

Источники новостей находятся в `space_astro_bot/sources.py`. Бот сначала пытается читать источник как RSS/Atom. Если RSS не найден, он пробует разобрать публичную HTML-страницу со списком материалов и сохранить найденные новости в SQLite.

По умолчанию включены:

- NASA News Releases;
- NASA Recently Published;
- NASA Artemis;
- ESA Space News;
- ESA Space Science;
- Sky & Telescope Astronomy News;
- Sky & Telescope Night Sky News;
- Российский космос — https://r-kosmos.ru/;
- Ин-Спейс — https://in-space.ru/;
- Космос-журнал — https://www.cosmos-journal.ru/;
- Новости Космонавтики — https://novosti-kosmonavtiki.ru/;
- Pro Космос — https://prokosmos.ru/.

Фото:

- NASA APOD API;
- NASA Image of the Day RSS.

## 7. Астрономические события

События хранятся в SQLite-таблице `astro_events`.

При старте бот загружает первичные события из:

```text
data/events.example.json
```

Бот показывает и рассылает только события в окне:

```text
сейчас <= событие <= сейчас + 7 дней
```

Логика уведомлений:

- один раз, когда до события осталось 1–7 дней;
- один раз, когда осталось меньше суток;
- один раз, когда осталось меньше часа;
- в момент события отправляет ссылку на трансляцию, если `livestream_url` заполнен.

## 8. Обновление старой версии

Если у тебя уже был запущен первый архив:

1. Останови бота в терминале через `Ctrl+C`.
2. Замени папку проекта на новую версию или скопируй новые файлы поверх старых.
3. Оставь свой `.env`, но проверь строку:

```env
BOT_TIMEZONE=Europe/Moscow
```

4. Запусти снова:

```bash
python -m space_astro_bot.main
```

Старая база SQLite должна обновиться автоматически: бот добавит новое поле для нескольких категорий.

## v4: автоматическое пополнение базы

В этой версии бот сохраняет найденные данные в SQLite-базу `space_astro_bot.sqlite3`, а не просто отправляет их пользователям.

Новые таблицы:

- `news_items` — архив новостей из RSS;
- `photo_items` — архив NASA APOD и NASA Image of the Day;
- `astro_events` — события, включая импорт из iCalendar;
- `sync_state` — время последней успешной синхронизации или последняя ошибка.

Автосинхронизация запускается по расписанию:

```env
NEWS_CHECK_INTERVAL_SECONDS=1800
PHOTO_CHECK_INTERVAL_SECONDS=21600
EVENT_SYNC_INTERVAL_SECONDS=21600
EVENT_CHECK_INTERVAL_SECONDS=1800
EVENT_IMPORT_DAYS=370
```

Что делают задачи:

- `news_job` — проверяет RSS, сохраняет новые новости в `news_items`, затем рассылает неподтвержденные новости подписчикам;
- `photos_job` — проверяет NASA APOD и NASA Image of the Day, сохраняет фото в `photo_items`, затем рассылает новые фото;
- `event_sync_job` — импортирует события из iCalendar-источников в `astro_events`;
- `events_job` — проверяет сохраненную базу `astro_events` и отправляет уведомления за неделю, сутки, час и в момент события, если есть ссылка на трансляцию.

По умолчанию для событий используется iCalendar-лента In-The-Sky.org:

```text
https://in-the-sky.org/newscalyear_ical.php?maxdiff=7&year={year}
```

Можно добавить свои iCalendar-источники в `.env`:

```env
EVENT_ICS_SOURCES=Мой календарь|https://example.com/calendar.ics;Еще календарь|https://example.com/events-{year}.ics
```

`{year}` автоматически заменяется на текущий и следующий год, если период импорта пересекает границу года.

Ручная синхронизация:

```text
/sync
```

Команда доступна администраторам. Если `ADMIN_CHAT_IDS` пустой, она доступна всем — это удобно для теста, но не подходит для публичного бота.

Проверить состояние базы можно командой:

```text
/status
```

## v6: перевод новостей и событий на русский

В этой версии бот может автоматически переводить новости и астрономические события на русский язык.

Перевод включается в `.env`:

```env
TRANSLATION_ENABLED=true
TRANSLATION_TARGET_LANG=ru
TRANSLATION_MAX_CHARS=3500
TRANSLATION_MAX_ITEMS_PER_RUN=20
```

Как это работает:

1. Бот получает новости из RSS/HTML-источников и события из iCalendar.
2. Оригинальный текст сохраняется в SQLite как `title`, `summary`, `description`.
3. Русский перевод сохраняется рядом в отдельных полях: `title_ru`, `summary_ru`, `description_ru`.
4. В Telegram пользователь видит русский вариант, а ссылка ведет на оригинальный источник.
5. Если переводчик недоступен, бот продолжает работать и показывает оригинальный текст.

Команда администратора:

```text
/translate
```

Она переводит уже сохраненные в базе свежие новости и события, у которых еще нет русского текста.

## Как сейчас обновляется база данных

Обновление базы выполняется автоматически через `JobQueue` в `space_astro_bot/main.py`:

```text
news_job       — проверяет новости каждые NEWS_CHECK_INTERVAL_SECONDS секунд
photos_job     — проверяет фото каждые PHOTO_CHECK_INTERVAL_SECONDS секунд
event_sync_job — импортирует события каждые EVENT_SYNC_INTERVAL_SECONDS секунд
events_job     — проверяет ближайшие события и рассылает напоминания
```

Ручное обновление запускается командой:

```text
/sync
```

После получения данных бот делает `upsert` в SQLite: если запись новая, она добавляется; если такая уже есть, она обновляется без создания дубля.
